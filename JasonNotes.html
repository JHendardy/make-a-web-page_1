The Web

-made up of HTML pages
-invented in early 1990s

Some Key Terms

HTML - hypertext markup language
URLs - uniform resource locator
HTTP - hypertext transfer protocol
Web Application - any software that runs in the web browser

HTML Tags & Elements

The structure of an element looks like this:

<tag attr="value"> contents </tag>

Anchor tags are used to include links. They look and work like this:

<a>
<a href="www.example.com">example</a>

Images can be displayed with the image tag. They are a void tag because they do not have closing tags. It's good to add an alt attribute to help if the image does not load and if people with impaired vision are visiting the site with a site reader. They look and work like this:

<img>
<img src="url" alt="text">

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Some examples of inline tags are: a, br, em, img, span

Some examples of block tags are: div, h1, p

Structure of HTML Page

The structure of an HTML page should look like:

<!DOCTYPE HTML>
<html>
<head>
</head>
<body>
</body>
</html>

In addition to this structure, structured code refers to the layout of code. It refers to the DOM, or the branches of the HTML.

Semantics within HTML is the practice of giving content on the page meaning and structure by using the proper element. Semantic code describes the value of content on a page, regardless of the style or appearance of that content.

Unit 1

Some Key Terms and Ideas

HTML - is used for structure
CSS - or cascading style sheets is used for style
DOM - or document object model refers to the HTML tree, which one can see in dev tools (such as Google's Developer Tools in Chrome)

- Everything is a box. It's helpful to break down webpages into seperate boxes or divs to help visualize the code.

- DIVs need classes to help label them. This will help when adding CSS to certain divs or elements.

- Avoid repetition to make your life easier.

- DRY = Don't Repeat Yourself. This is important and helpful for organization and readable code. By being organized and not repeating code it's easier to edit and figure out an issue with a project.

- Group reusable css properties together
- Name these groups logically
- Add your selectors to the various css groups

Some CSS Things to Know

- If you want to select a class you can use ."classname" in the CSS sheet. For IDs use #.

Margin - outer layer
Border - border inside margin but around padding
Padding - gutter around content
Content - very center

Since DIVs are block elements (as opposed to inline), by default they take up the entire width of a page.
- Adding the rule display: flex to the appropriate CSS wil override this behavior and let DIVs apppear next to each other

box-sizing: border-box

- Makes it so the element size include border and padding, size of the box will always stay the same when border and padding is tweaked. Kind of like holding SHIFT in PSD.
- Because it's new you need to make sure browsers are compatible.

For graceful degradation one must account for all browsers:

-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
box-sizing: border-box;
